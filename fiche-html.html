Tout ce qu'il faut savoir pour l'examen — en français simple !

🏗️ 1. La structure de base d'une page HTML

Toute page HTML commence toujours avec cette structure :

<!doctype html> ← dit au navigateur : "c'est du HTML5" <html lang="fr"> ← langue de la page <head> ← infos invisibles (titre, encodage...) <meta charset="UTF-8" /> ← accents et caractères spéciaux <meta name="viewport" content="width=device-width" /> <title>Titre dans l'onglet</title> </head> <body> ← tout ce que l'utilisateur voit ... </body> </html>
💡 Astuce : <head> = invisible (pour le navigateur), <body> = visible (pour l'utilisateur) !

🧠 2. Balises sémantiques vs génériques

Une balise sémantique a un sens, elle décrit ce qu'elle contient. Une balise générique ne veut rien dire.

BaliseSémantique ?C'est quoi ?
<header>✅ OUIEn-tête de la page
<nav>✅ OUIMenu de navigation
<main>✅ OUIContenu principal (1 seul par page !)
<section>✅ OUISection thématique
<article>✅ OUIContenu indépendant (blog, news...)
<aside>✅ OUIContenu secondaire / latéral
<footer>✅ OUIPied de page
<div>❌ NONBoîte générique (block)
<span>❌ NONBoîte générique (inline)
⚠️ <aside> et <ul> sont les MOINS appropriés pour la navigation principale !

✍️ 3. Balises de texte

✅ Sémantiques (avec sens)

  • <h1> à <h6> — Titres
  • <p> — Paragraphe
  • <strong> — Important (gras)
  • <em> — Emphase (italique)

❌ Génériques (juste visuel)

  • <b> — Gras sans sens
  • <i> — Italique sans sens
  • <br /> — Saut de ligne
💡 Retiens : <strong> = important (pour les lecteurs d'écran aussi), <b> = juste du gras visuel.

🔒 4. Balises auto-fermantes

Ces balises n'ont pas de contenu donc elles se ferment toutes seules avec /> :

<img src="photo.png" alt="description" /> <input type="text" /> <br /> <meta charset="UTF-8" /> <link rel="stylesheet" href="style.css" />

🖼️ 5. Images et accessibilité (RGAA)

L'attribut alt est toujours obligatoire sur une image. Sa valeur dépend du type d'image :

Type d'imageValeur de altExemple
🖼️ Image informativeDescription précisealt="Logo de l'entreprise"
🎨 Image décorativeVide ("")alt=""
🔗 Image-lienDestination du lienalt="Retour à l'accueil"
❌ Sans alt—TOUJOURS INVALIDE !
← Image informative <img src="logo.png" alt="Logo de l'entreprise Dupont" /> ← Image décorative <img src="deco.png" alt="" /> ← Image sans alt = INVALIDE ! <img src="photo.png" /> ❌

🔗 6. Les liens — balise <a>

← Lien vers une page <a href="page.html">Cliquez ici</a> ← Lien vers une section de la même page <a href="#contact">Aller à Contact</a> ← Lien email <a href="mailto:test@mail.com">Envoyer un email</a> ← ERREUR : src n'existe pas sur <a> ! <a src="page.html">Lien</a> ❌
⚠️ <a> utilise href (pas src !). src c'est pour <img> et <script>.

📝 7. Les formulaires

<form action="/signup.php" method="post"> <label>Nom d'utilisateur</label> <input type="text" name="username" required placeholder="Votre nom" /> <label>Mot de passe</label> <input type="password" name="password" required /> <input type="submit" value="Envoyer" /> </form>
AttributCe que ça fait
actionOù envoyer les données
method="get"Données dans l'URL (visible ! Jamais pour les mots de passe)
method="post"Données cachées ✅ (pour mots de passe)
requiredChamp obligatoire
placeholderTexte indicatif grisé
disabledChamp désactivé

✅ Types d'input valides

  • type="text" — Texte libre
  • type="email" — Valide le format
  • type="password" — Cache les caractères
  • type="number" — Chiffres
  • type="checkbox" — Case à cocher
  • type="submit" — Bouton d'envoi

❌ N'existent PAS

  • type="secure" ❌
  • Attribut optional ❌
  • Attribut src sur <a> ❌
  • Balise <row> ❌
  • Balise <tcontent> ❌

📋 8. Les listes

← Liste à puces <ul> <li>Pomme</li> <li>Banane</li> </ul> ← Liste numérotée <ol> <li>Premier</li> <li>Deuxième</li> </ol>
← Liste de définitions <dl> <dt>HTML</dt> <dd>Langage du web</dd> </dl>
⚠️ Un <li> doit TOUJOURS être dans un <ul> ou <ol> !
